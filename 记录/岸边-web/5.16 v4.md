招聘数据已接入 SQLite + Prisma，主要改动如下。

## 1. 数据库模型 `prisma/schema.prisma`

已追加 `Job` 表，字段为：`id`、`title`、`deadline`、`majors`、`slots`、`createdAt`。

## 2. 发布 API `app/api/jobs/route.js`

- 使用 `PrismaClient`（在 `lib/prisma.js` 中实例化）
- `POST /api/jobs` 写入：`prisma.job.create({ data: { title, deadline, majors, slots } })`
- `slots` 经 `parseInt` 转成整数后再入库

## 3. 管理后台 `app/admin/page.js`

发布改为请求 `/api/jobs`，字段为：`title`、`deadline`、`majors`、`slots`（与数据库一致）。

## 4. 首页 `app/page.js`

- 服务端：`prisma.job.findMany({ orderBy: { createdAt: 'desc' } })`
- 已去掉 mock / localStorage
- 筛选 UI 在 `app/home-client.js`

## 5. 辅助文件

- `lib/prisma.js`：Prisma 单例（Prisma 7 + SQLite adapter）
- `lib/job-utils.js`：卡片展示映射（城市、分类、倒计时等）

## 你需要在终端执行的命令

若还没跑过迁移（我们这边已跑过一次），在项目根目录执行：

cd "d:\工作相关\anBian-web"

npx prisma migrate dev --name add_job_model

这条命令会：

1. 根据 `schema.prisma` 生成/更新 SQLite 文件（默认是项目根目录的 `dev.db`）
2. 执行迁移 SQL，创建 `Job` 表
3. 自动生成 `@prisma/client`

仅重新生成客户端时：

npx prisma generate

确认 `.env` 里有（你已有）：

DATABASE_URL="file:./dev.db"

然后重启开发服务：

npm run dev

## 如何验证

1. 打开 [http://localhost:3000/admin](http://localhost:3000/admin) → 提炼 → 一键发布
2. 打开 [http://localhost:3000](http://localhost:3000/) → 应看到刚发布的岗位（按发布时间倒序）
3. 可用 `npx prisma studio` 在浏览器里直接查看 `Job` 表数据

> 说明：Prisma 7 需要 SQLite adapter，已安装 `@prisma/adapter-better-sqlite3` 和 `better-sqlite3`。若换机器部署，需重新执行 `npm install` 和 `npx prisma migrate dev`。
